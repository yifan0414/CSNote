目前在 DYTO 的两个阶段分别引入 Question-aware 进行了实验验证（在 Egochema 数据集上）
**方案 A：question-aware frame selection**（基于 CLS）
做法：用 CLIP(Q, frame) similarity
结果：47.2( <font color=" #c00000 ">-0.8%</font>)
可能得问题：在初始选帧阶段过度聚集问题相关的局部，**可能破坏了全局事件结构**。这与 Egoschma 数据集的特性有关（测试模型能否把长视频抽象成“事件结构+因果关系”）

**方案 B：question-aware token merging**
做法：在聚类时将帧间的距离公式更改为 $d(i,j)=α⋅⟨v^i​,v^j​⟩+(1−α)⋅∣sim(Q,i)−sim(Q,j)∣$
结果：50.4(<font color=" #c00000 ">+2.4%</font>)
原因：让 cluster 更“语义对齐问题”，减少无关事件

虽然方案 B 有提点，但距离目前的 **SOTA（D-code）58.0%** 仍有差距。这可能和 Egoschema 这个数据集有关，DYTO 的 pipeline 如下，不论怎么选帧，都是单步推理，Egoschema 的瓶颈并不是“选对帧”，而是“看到了，但不知道如何推理”

```
DYTO pipeline
Video → frames → tokens → LLM(VideoTokens + Q) → Answer
-----------------------------------------------------------------
D-code pipeline
Video → frames → tokens
Q → {Q1, Q2, Q3, ...}
LLM(VideoTokens + Qi) → Ai
LLM(VideoTokens + {A1...An} + Q) → Final Answer
```
D-Code 在选帧阶段并没有引入 Question-aware，而是对于同一组 video tokens，多轮独立推理（在时间/因果/对比子问题方面）。从而实现了 <font color=" #c00000 ">+6.2%</font> 的巨大提升（消融实验中引入子问题分解）。

| 模型       | 信息覆盖 | 推理结构 | Egochema |
| -------- | ---- | ---- | -------- |
| IG-VLM   | ❌    | ❌    | 35.8     |
| SlowFast | ⚠️   | ❌    | 47.2     |
| DYTO     | ✅    | ❌    | 48.0     |
| D-CoDe   | ⚠️   | ✅    | 58.0     |
D-CoDe 的选帧并没有 DYTO 全面，但对于 Egochema 这类任务，瓶颈不在帧-问题的覆盖率，而可能在“已有的帧能否被合理的消费推理”，是推理瓶颈，不是感知瓶颈。
目前我想的是，更好的感知（DTYO 选帧）+更好的推理（D-CoDe 子问题分解）可能会更有优势，但 D-CoDe 的代码并没有开源（作者回复 12 月底发布也没有发布）。
