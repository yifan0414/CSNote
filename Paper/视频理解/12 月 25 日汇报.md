好，我们把“DYTO / SlowFast-LLaVA 这条 **image-based MLLM + zero fine-tuning**”路线，落到 **EgoSchema** 上来具体评估你说的 3 个落点：**创新性、与现有工作的重叠风险、顶会可接受性**。

先对齐一下 EgoSchema 的“题型压力点”：

* EgoSchema 是**超长时序理解诊断型**数据集：多选题（5 选 1），基于约 **3 分钟**片段，但用 *temporal certificate sets* 衡量的“内在时序长度”非常长；现有 SOTA 仍显著低于人类（论文里报告人类约 76%，不少开源模型 <33%，随机 20%）。([arXiv][1])
  这意味着你在 EgoSchema 上要赢，关键通常不是“更细的单帧识别”，而是：**抓住稀疏但决定性的事件 + 维持跨时间的因果/状态链**。

---

## 落点 1：把“选帧/选片段”和“token merging”做成统一的预算分配器（Selection–Compression Unification）

### 你能做出什么“新东西”

DYTO 已经是“分层选帧 + bipartite token merging”的强基线。([arXiv][2])
但它仍更像 **两段式 pipeline**：先挑时段/帧，再合并 token。你的创新空间在于做成 **一个统一的决策问题**：

* 决策变量不止“选哪些帧”，还包括：**每一段分配多少 token、每帧合并到什么粒度、是否改用 clip/panel 单元**（例如把“动作连续性”用 clip 保住）。([arXiv][3])
* 算法上可以借鉴“严格预算下探索-利用”的思想（FOCUS 把选择写成 CPE bandit，并有理论保证）。([arXiv][4])
  你可以提出：**Budgeted Spatio-Temporal Token Allocation**（完全不训练），输出的是“时间段×合并粒度”的联合计划，而不是单纯的帧列表。

### 会不会和别人重复

**中等风险**。因为 2025 年的趋势就是把“选择+压缩”往一起推：

* D-CoDe 也是“dynamic compression（代表帧 + spatial token 聚合）”并配合 question decomposition。([arXiv][5])
* 选择派也越来越强（DIG：先判 global/local 再走不同选择策略；AdaRD-Key：相关性+多样性联合目标；F2C：从帧到 clip）。([arXiv][6])

**怎么避免撞题：**

1. 明确你做的是 **“联合优化/联合调度”**，而不是再提一个新选帧得分函数。
2. 做出 **Pareto 曲线**：准确率 vs 视觉 token 数/延迟/显存，而不是只报准确率。EgoSchema 对“抓关键事件”很敏感，这种 trade-off 展示会很有说服力。
3. 设计 **EgoSchema 特化的诊断**：按 temporal certificate 长度分桶，证明你的“联合预算”在长证书桶提升最大（这点 reviewers 很吃）。([arXiv][1])

### 顶会接受可能性（我会评为：偏高）

原因：

* 这是“机制层”的贡献（budget allocation + unify），容易写成清晰的算法贡献；
* EgoSchema 本身“很难”且被广泛认可，能在这上面稳定涨点很硬。([arXiv][1])
  前提是你必须给出：**强 ablation + 复杂度分析 + 误差分析（哪些题型改善）**。

## 落点 2：系统性诊断并校正 image-MLLM 在视频序列上的注意力/位置偏置（Bias Diagnosis & Correction）

### 你能做出什么“新东西”

这条线的关键是：把“偏置”从一个口号变成**可量化、可复现、可解释**的现象，然后给出 **训练免费的校正**。

注意：LLaVA-MLB 已经明确在做“attention bias 的缓解与利用”（提出 Gridded Attention Pooling + Visual Summarization Tail）。([arXiv][7])
所以你不能再做一个“也解决 attention bias”的同质化方法。

**你在 EgoSchema 上的差异化机会：**

* EgoSchema 有 temporal certificate 的概念，你可以定义 **Bias–Certificate coupling**：

  * 当证书长度更长时，模型是否更容易出现“**首帧/末帧偏置**”、“**近因效应**（只看后面）”、“**视觉重复导致的注意力塌缩**”？
* 给出一个**不训练**的校正器，例如：

  1. **Order-invariant pooling**：同一组帧用多种顺序喂入（或分块重排）得到多个 logit，再做一致性融合；
  2. **Bias-calibrated scoring**：对注意力分布做温度校准/去趋势（detrending），让“帧的重要性”不被位置先验支配；
  3. **Certificate-aware resampling**：当偏置指标触发时，强制补采样“被忽视的时间段”。

这些都可以严格保持“no fine-tuning”，且会非常贴合 EgoSchema 的诊断属性。([arXiv][1])

### 会不会和别人重复

**较高风险**（因为 LLaVA-MLB 已经占了“attention bias”这个坑）。([arXiv][7])

**怎么避免撞题：**

* 你把核心贡献放在 **“EgoSchema 的偏置诊断基准 + 证书长度分解分析”**，校正方法保持简单但有效；
* 或者你做的是 **“跨方法统一的偏置度量 + 一键校正”**，能同时改善 DYTO / SlowFast-LLaVA / D-CoDe 这类 pipeline（作为通用 plug-in）。([arXiv][2])

### 顶会接受可能性（中等）

如果你只是“提出一个新 pooling”，容易被认为 incremental。
如果你能做到：**新诊断协议 + 明确现象 + 很强的 cross-model/cross-setting 结论**，那就有机会冲 CVPR/ICCV/NeurIPS（更像“分析+方法”的组合贡献）。

## 落点 3：让 query decomposition 真正驱动“证据收集”，并利用 EgoSchema 的多选结构做“选项对比式证据检索”（Option-Contrast Evidence Gathering）

### 你能做出什么“新东西”

D-CoDe 已经做了 question decomposition 来缓解 token overload。([arXiv][5])
所以你如果只做“把问题拆成子问题再问”，重复度很高。

EgoSchema 的优势是它是 **5 选 1 多选题**：这天然适合做一种更有辨识度的 training-free 策略：

* **Option-as-Query**：把每个候选答案 (a_i) 和问题 (q) 拼成一个“假设查询” (q \oplus a_i)，分别去选帧/选 clip/分配 token；
* 然后做 **对比式证据**：找出“支持正确选项、反驳干扰项”的时间片段（类似检索式反证）。
  这比“分解问题”更贴合多选任务，也更可能带来 EgoSchema 的实增（因为干扰项常常需要靠长时序细节排除）。

你还能把 DIG 的“先判 query 类型”接进来：

* global 型问题：对所有选项共享一套粗采样；
* localized 型问题：对每个选项触发更精细的证据收集。([arXiv][6])

### 会不会和别人重复

**中等风险**。因为“decomposition/iterative reasoning”已经很多，但“**选项对比式证据收集**”在这条 *image-MLLM + training-free* 视频理解分支里相对更少被当成主贡献（至少在 D-CoDe 的摘要层面，它强调的是 decomposition 来分摊 token，而不是 MC 结构化对比）。([arXiv][5])

**怎么避免撞题：**

* 把你的贡献写成：**面向多选长视频 QA 的 training-free “contrastive evidence” 框架**（而不是泛化的 decomposition）。
* 做强诊断：哪些错题从“看错”变成“排除干扰项”，给出可视化证据片段（reviewers 很买账）。

### 顶会接受可能性（偏高）

如果你在 EgoSchema 上能拿到稳定提升，并且能证明它是“多选结构 + 长证书”共同导致的增益，那么这条线很像“任务结构化 + 推理编排”的贡献，投 EMNLP/NeurIPS 都有合理叙事空间（尤其是你强调 reasoning 与证据链）。

## 结合 EgoSchema，我会怎么建议你选主线

如果你要冲顶会、且尽量避开现有 2025 工作的“主战场拥挤区”，我建议优先级是：

1. **落点 3（Option-contrast evidence）**：最贴 EgoSchema，多选结构天然 novelty，和 D-CoDe 的“分解”容易拉开叙事差异。(^[5])
2. **落点 1（统一预算分配）**：工程与算法都硬，最容易做出漂亮的 Pareto 曲线 + 复杂度优势，竞争力强。([arXiv][2])
3. **落点 2（偏置诊断校正）**：要做得足够“研究味”才能赢过 LLaVA-MLB 的先发优势，更适合作为 1/3 的增强模块或附加贡献。([arXiv][7])
