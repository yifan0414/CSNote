最近在看 Mixture-of-Experts（MoE）的相关论文，因为我发现上次组会汇报的那 10 篇做 training-free 的 video understand 论文，例如 DyTo\PruneVid\FitPrune 基本上都是围绕 Token 剪枝、关键帧选择上做工作，面向众多 Video QA 数据集的通用方法（也意味着是单一策略），即使加入问题内容感知的 Free Video-LLM 和 DtoMe，**也只是在 token 级别做自适应的合并（并且算不上多智能，只使用了 CLIP 计算相似度）**。

而 MoE 与 test time router 这个思想目前也有工作和进展，我列举两个本领域最相关的论文
TimeExpert: An Expert-Guided Video LLM for Video Temporal Grounding, ICCV 2025
1. 做的是 Video Temporal Grounding / Dense caption / Highlight detection；
2. 核心是一个 MoE-based Video-LLM：把时间戳 token、显著性 token、caption token 路由给不同专家；
3. 这个是需要训练的

CoT-Vid: Dynamic Chain-of-Thought Routing with Self Verification for Training-Free Video Reasoning（arXiv 2025.05.11）⭐️（没看到这篇论文之前，我也在考虑在推理层面做 test-time 路由）
1.  training-free video reasoning 范式
2.  核心是 dynamic chain-of-thought routing 也就是在不同推理流程 / CoT 路径之间做动态选择和验证。
3. 位于推理层面，也就是 Dyto\PruneVid 等方法的下层
4. 强依赖于 prompt 工程

基于上述，我想结合两者提出一个新的思路，位于token/帧策略的动态路由与分配。
**解决什么问题**：如何在有限的 token 预算下，获得最优的信息（token 预算怎么用的更聪明）
**位于什么层次**：感知/token 层
**方法**：例如定义一族视频 token 路由策略（ToMe/DToMe/DyTo）的各类聚类方法，考虑视觉文本下的 token 的预算。在对这些策略使用 test-time 的相关自评分标准（自一致性，成本约束）等选出一个最优的策略。
